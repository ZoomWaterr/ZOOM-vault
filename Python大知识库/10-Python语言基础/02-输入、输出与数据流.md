# 02 输入、输出与数据流

刷题程序的骨架永远是三段：

```text
输入 → 处理 → 输出
```

题目不会主动把数据变成你想要的样子。你要先把输入读对。

---

## input 读到的永远是字符串

```python
s = input()
```

即使你输入的是 `10`，读进来也还是字符串 `"10"`。要计算，就要转换：

```python
n = int(input())
```

---

## 一行多个整数

```python
a, b = map(int, input().split())
```

这行代码可以拆开理解：

```text
input()       读一整行
split()       按空格拆开
map(int, ...) 每一项转整数
a, b = ...    分别接住
```

---

## 一行整数列表

```python
nums = list(map(int, input().split()))
```

这是你现在最该自动化的写法之一。不要每次重新想，把它练到手能直接打出来。

---

## 输出列表

OJ 通常要的是：

```text
1 2 3
```

不是：

```text
[1, 2, 3]
```

所以要写：

```python
print(*nums)
```

或者：

```python
print(" ".join(map(str, nums)))
```

---

## 小结

输入输出不是低级内容。基础题里大量错误都发生在这里：读成了字符串、读错了行数、输出多了括号、少了小数位。先把数据流稳定下来，算法才有意义。
