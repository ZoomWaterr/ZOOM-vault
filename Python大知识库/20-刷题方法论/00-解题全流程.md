# 🧭 解题全流程 · 从读题到 AC

> 拿到一道题，不知道从哪里下手，就按这 5 步走。

## 五步框架

```text
STEP 1  读懂题意
- 用大白话复述：这题到底要我干嘛？
- 圈出关键数字：n 最大多少？有负数吗？要取模吗？
- 样例先手算一遍。

STEP 2  建立模型
- 输入是什么？
- 我要维护什么变量？
- 输出是什么？

STEP 3  匹配方法
- 统计次数 → 字典
- 连续一段 → 前缀和 / 双指针
- 区间批量操作 → 差分
- 当前依赖前面 → 递推 / DP

STEP 4  写代码
- 先写输入输出骨架。
- 再写核心逻辑。
- 最后处理边界。

STEP 5  验证
- 样例。
- n=1。
- 全相同 / 全不同 / 极端值。
```

## 关键提醒

卡住时问“我现在最基本要做什么”，这个问题很好。它比“这题应该用什么高级算法”更接近真实入口。
