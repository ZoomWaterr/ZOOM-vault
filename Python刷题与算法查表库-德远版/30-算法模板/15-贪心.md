# 贪心

## 快速结论

贪心每一步做当前最优选择，但必须保证这个选择不会破坏全局最优。

## 什么时候用

- 排序后选择。
- 区间选点。
- 最少数量覆盖。
- 排队等待时间最小。
- 局部最优很自然，并且能解释为什么不会后悔。

## 区间调度模板

选择最多不重叠区间：按结束时间排序。

```python
intervals.sort(key=lambda x: x[1])
ans = 0
last_end = -10**18

for l, r in intervals:
    if l >= last_end:
        ans += 1
        last_end = r

print(ans)
```

## 排队等待时间

短任务优先。

```python
times.sort()
waiting = 0
prefix = 0

for t in times:
    waiting += prefix
    prefix += t
```

## 贪心判断

写贪心前问：

1. 我每一步选的是什么？
2. 为什么这个选择不会让后面变差？
3. 能不能构造反例？

## 复杂度

通常排序 `O(n log n)`，扫描 `O(n)`。

## 易错点

- 排序规则错，贪心就错。
- 看起来对不代表真的对，要尝试反例。
- 不确定时和 DP 对比：DP 记录状态，贪心不记录历史。
