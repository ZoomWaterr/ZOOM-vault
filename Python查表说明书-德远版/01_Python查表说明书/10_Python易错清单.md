# Python 易错清单

tags: #python/易错 #刷题/检查

## 写完先扫一遍

- `input()` 是字符串，算数前转 `int`。
- `/` 得到 float，整数除法用 `//`。
- `sort()` 改原列表，`sorted()` 返回新列表。
- `join()` 只能拼字符串列表。
- `{}` 是空字典，空集合是 `set()`。
- `s[:-2]` 是切到倒数第二个之前，不是反向步长。
- 二维数组不要用“同一行列表乘法复制 n 次”的写法，会共用同一行。
- 遍历字典时想拿 key 和 value，用 `for k, v in d.items()`。
- `rstrip(".")` 是去右端连续的点，不是清理所有标点。
- 循环里修改下标时，想清楚边界和 `+1` 会不会越界。

## 我的高危区

1. 嵌套循环索引混乱：外层 `i`、内层 `j` 容易打架。
2. 数学题一不清楚就退回暴力。
3. 样例过了就提交，不主动测边界。
4. 注释掉的旧代码堆在文件里，复习时干扰判断。
5. 平铺代码太长，bug 不好定位。
