**快速结论**
DP 用已经算过的小问题，推出更大的问题。

**什么时候用**
- 求最优值。
- 求方案数。
- 当前结果依赖前面的状态。
- 有重复子问题。
- 题目出现“选或不选”。

**写 DP 前先回答四个问题**
1. 状态是什么：`dp[i]` 表示什么？
2. 选择是什么：当前位置有哪些选择？
3. 初始值是什么：base case？
4. 转移是什么：`dp[i]` 从哪里来？

**爬楼梯模板**

```python
n = int(input())

if n <= 1:
    print(1)
else:
    dp = [0] * (n + 1)
    dp[0] = 1
    dp[1] = 1

    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]

    print(dp[n])

```

**最大子数组和**

```python
dp = [0] * len(nums)
dp[0] = nums[0]
ans = nums[0]

for i in range(1, len(nums)):
    dp[i] = max(nums[i], dp[i - 1] + nums[i])
    ans = max(ans, dp[i])

```

**空间优化**
如果 `dp[i]` 只依赖前几个状态，可以只保留变量。

```python
prev2, prev1 = 1, 1
for i in range(2, n + 1):
    cur = prev1 + prev2
    prev2, prev1 = prev1, cur

```

**复杂度**
`O(状态数 * 每个状态的转移数)`。

**易错点**
- `dp[i]` 的含义说不清，就不要写代码。
- 初始值最容易错。
- 遍历顺序必须保证依赖状态已经算过。
