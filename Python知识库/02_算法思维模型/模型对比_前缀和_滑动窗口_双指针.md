# 模型对比_前缀和_滑动窗口_双指针

这三个都经常处理数组或字符串里的“位置关系”，但信号不同。

## 一句话区别

- 前缀和：快速求固定区间的累计值。
- 滑动窗口：维护连续区间并动态扩缩。
- 双指针：两个位置协同移动，减少枚举。

## 对比表

| 模型 | 关键词 | 典型问题 |
|---|---|---|
| 前缀和 | 区间和、多次查询 | `[l,r]` 求和 |
| 滑动窗口 | 连续子串、最长/最短 | 最长不重复子串 |
| 双指针 | 有序、两端、两个数 | 有序数组两数之和 |

## 什么时候用前缀和

题目多次问：

```text
某一段的和是多少？
```

## 什么时候用滑动窗口

题目问：

```text
满足条件的最长/最短连续一段
```

## 什么时候用双指针

题目有：

```text
有序数组
两个下标
左右向中间靠
```

## 易错点

- 前缀和不负责动态移动窗口。
- 滑动窗口通常要求连续。
- 双指针常常依赖有序性。
