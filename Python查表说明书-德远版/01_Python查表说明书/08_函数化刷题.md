# 函数化刷题

tags: #python/函数 #刷题/习惯

## 应该怎么做

刷题不一定写工程代码，但核心逻辑至少可以抽成一个函数。

```python
def solve():
    n = int(input())
    nums = list(map(int, input().split()))
    print(核心逻辑(nums))


def 核心逻辑(nums):
    return sum(nums)


if __name__ == "__main__":
    solve()
```

## 这在干什么

函数的作用不是装高级，是给大脑减负。输入、处理、输出拆开，错了就知道去哪里看。

## 我的提醒

诊断报告里说得很准：你不是不会写函数，你的 `stats.py` 结构很清楚；你只是刷题时觉得“没必要”。但函数会让调试更快。

## 什么时候翻它

- 题目逻辑超过 20 行。
- 你开始在一大坨代码里找 bug。
- 同一段逻辑可能复用：质数、回文、GCD、读入处理。
