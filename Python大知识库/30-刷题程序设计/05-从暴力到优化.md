# 05 从暴力到优化

暴力解法不是低级，它是理解题意的第一稿。

真正的问题是：当数据变大时，暴力还来不来得及？

---

## 常见优化方向

| 暴力在做什么 | 优化方向 |
|:---|:---|
| 每次重新求一段和 | 前缀和 |
| 每次修改区间里每个数 | 差分 |
| 每次查一个东西出现过没有 | 集合/字典 |
| 重复计算前面的状态 | 递推/动态规划 |
| 有序数据里线性找 | 二分 |

---

## 什么时候先写暴力

- 数据范围很小。
- 你还没完全读懂题。
- 需要先验证规律。

暴力能帮助你理解题，但不要让它成为永远的终点。

---

## 小结

优化的核心是减少重复劳动。每次想优化，都问一句：我是不是把同一件事做了太多遍？
