# 从暴力到优化

tags: #刷题/优化 #算法/思维

## 应该怎么做

先写出能表达题意的暴力，再问它慢在哪里。

## 优化方向

| 暴力慢在哪里 | 常见优化 |
|---|---|
| 每次都重新求区间和 | 前缀和 |
| 每次都修改一整段 | 差分 |
| 反复在线性查找有没有 | 字典/集合 |
| 有序后还一个个找 | 二分 |
| 重复算同样子问题 | DP/记忆化 |
| 每次找最大最小 | 堆 |
| 枚举所有连续区间 | 滑动窗口/前缀和 |

## 数学题流程

```text
手算小例子 -> 观察规律 -> 写公式 -> 验证 -> 代码实现
```

## 我的提醒

你退回暴力不是坏事，坏的是退回后不再往前看。暴力是理解题意的脚手架，不一定是最终答案。
