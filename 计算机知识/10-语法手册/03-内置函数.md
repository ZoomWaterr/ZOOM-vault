## 求和 / 个数 / 平均

```python
sum(a)                  # 列表求和，元素必须数字
len(a)                  # 元素个数，所有容器通用
sum(a) / len(a)         # 平均数 → 返回float
sum(a) // len(a)        # 整数平均
sum(i**2 for i in range(n))  # 生成器求和，省内存
```

## 四舍五入 / 保留小数

> `round` 和 f-string 都能控制小数，但**行为不同**。

### round(x, n)

```python
round(3.1415, 2)    # → 3.14
round(7, 2)         # → 7      ⚠️ 不补零！
round(3.1415)       # → 3     不写n → 四舍五入到整数
```

### f-string 格式化

```python
f"{3.1415:.2f}"     # → "3.14"
f"{7:.2f}"          # → "7.00"   ✅ 会补零！
f"{3.1415:.6g}"     # → "3.1415"  6位有效数字，自动删末尾0
```

### 什么时候用哪个？

| 场景 | 用什么 | 原因 |
|:---|:---|:---|
| OJ要求保留2位小数 | `f"{x:.2f}"` | 会补零，稳定 |
| 中间计算需要舍入 | `round(x, 2)` | 数值层面 |
| OJ要求输出整数 | `int(x)` 或 `round(x)` | 看是截断还是舍入 |

> 🔥 **最常见WA**：`round(7, 2)` 输出 `7`，OJ要 `7.00`。用 `f"{7:.2f}"`。

## 最大 / 最小

```python
max(a)                      # 普通比较
min(a)

# 按自定义标准 — key= 不是关键字名，是「按什么比」
max(words, key=len)         # 找最长字符串
max(count, key=count.get)   # 找value最大的key
max(nums, key=lambda x: abs(x))  # 找绝对值最大
```

## 排序

```python
sorted(a)                       # 返回新列表，原列表不变
sorted(a, reverse=True)         # 从大到小
sorted(a, key=len)              # 按长度排

# sorted vs list.sort()
# sorted() → 返回新列表，不改原列表  ← 更安全
# a.sort() → 直接改原列表，返回None
```

## 遍历辅助

```python
# enumerate — 同时拿下标和值
for i, x in enumerate(lst):
    print(f"第{i}个是{x}")

# zip — 同时遍历多个列表
for name, score in zip(names, scores):
    print(f"{name}: {score}")

# range
range(5)           # → 0,1,2,3,4
range(2, 8)        # → 2,3,4,5,6,7
range(0, 10, 2)    # → 0,2,4,6,8
```

## map / filter

```python
# map：对每个元素应用函数 — 输入必用
a, b = map(int, input().split())
nums = list(map(int, input().split()))

# filter：筛出满足条件的 — 推导式更直观
list(filter(lambda x: x > 0, [-1, 0, 1, 2]))  # → [1, 2]
# 等价于：[x for x in [-1, 0, 1, 2] if x > 0]
```

## 💬 德远的笔记

> `max(count, key=count.get)` 第一次见以为是天书——其实 `key=` 就是「按什么标准比大小」。`count.get` 是取 key 对应的 value。翻译：**找 value 最大的那个 key**。
>
> OJ 输出小数用 f-string 别用 round。`round(7, 2)` 输出 `7` 不是 `7.00`，WA 了才后悔。
