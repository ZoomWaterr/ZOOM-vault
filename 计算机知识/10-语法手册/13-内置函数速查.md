> `sum()`、`round()`、`abs()`、`max()`、`min()`、`len()`、`sorted()`……

## 求和 / 求平均 / 个数

| 我想…… | 代码 | 注意 |
|:---|:---|:---|
| 列表求和 | `sum(a)` | 元素必须是数字 |
| 列表元素个数 | `len(a)` | 所有容器通用 |
| 求平均数 | `sum(a) / len(a)` | 返回 float；整数平均用 `sum(a) // len(a)` |
| 从生成器求和 | `sum(i**2 for i in range(n))` | 不需要先建列表，省内存 |

## 四舍五入 / 保留小数

> round 和 f-string 都能控制小数位数，但**行为不同**。

### round(x, n) — 四舍五入保留 n 位

```python
round(3.1415, 2)    # → 3.14
round(3.1415, 0)    # → 3.0    ⚠️ 返回 float，不是 int
round(7, 2)         # → 7      ⚠️ 不会补零！不是 7.00
```

### f-string 格式化 — 控制显示

```python
f"{3.1415:.2f}"     # → "3.14"
f"{7:.2f}"          # → "7.00"   ✅ 会补零！
f"{3.1415:.6g}"     # → "3.1415" 6 位有效数字，自动删末尾 0
```

### 什么时候用哪个？

| 场景 | 用什么 | 原因 |
|:---|:---|:---|
| OJ 要求输出保留 2 位小数 | `f"{x:.2f}"` | 会补零，输出稳定 |
| 中间计算需要四舍五入 | `round(x, 2)` | 数值层面的舍入 |
| OJ 要求输出整数 | `int(x)` 或 `round(x)` | 取决于是截断还是舍入 |

> 🔥 **最常见的坑**：OJ 要输出 `7.00`，你写 `round(7, 2)`，输出 `7` — WA！用 `f"{7:.2f}"` 保平安。

## 最大 / 最小

```python
max(a)              # 普通比较
min(a)              # 同上

# 按自定义标准找最大/最小
max(words, key=len)           # 找最长的字符串
max(count, key=count.get)     # 找出现次数最多的 key
max(nums, key=lambda x: abs(x))  # 找绝对值最大的数
```

> `key=xxx` 不是「关键字参数的名字」，而是「按什么标准比较」。`max(a, key=len)` 翻译：找 a 里**按长度算**最大的那个。

## 排序

```python
sorted(a)                  # 返回新排序列表，原列表不变
sorted(a, reverse=True)    # 从大到小
sorted(a, key=len)         # 按长度排
sorted(a, key=lambda x: x[1])  # 按第二个元素排
```

和 `a.sort()` 的区别：`sorted()` **不修改原列表**，返回新列表。`a.sort()` **直接改原列表**，返回 None。

## 类型转换

```python
int("123")       # → 123    字符串→整数
int(3.9)         # → 3      ⚠️ 截断，不是四舍五入！
float("3.14")    # → 3.14   字符串→浮点数
str(123)         # → "123"  整数→字符串
bool(0)          # → False  0、空串、空列表为 False，其余 True

# input() 返回的是字符串，必须转换！
n = int(input())
nums = list(map(int, input().split()))
```

> 🔥 `int()` 是截断不是四舍五入！`int(3.9) → 3`，`int(-3.9) → -3`。四舍五入用 `round()`。

## 遍历辅助

```python
# enumerate — 同时拿下标和值
for i, x in enumerate(lst):
    print(f"第 {i} 个是 {x}")

# zip — 同时遍历多个列表
for a, b in zip(lst1, lst2):
    print(a, b)

# range — 生成数字序列
range(5)           # → 0,1,2,3,4
range(2, 8)        # → 2,3,4,5,6,7
range(0, 10, 2)    # → 0,2,4,6,8
```

## map / filter

```python
# map：对每个元素应用同一个函数
list(map(int, ["1", "2", "3"]))    # → [1, 2, 3]

# 输入必用
a, b = map(int, input().split())
nums = list(map(int, input().split()))

# filter：筛出满足条件的
list(filter(lambda x: x > 0, [-1, 0, 1, 2]))  # → [1, 2]

# 但说白了：能用推导式的地方用推导式更直观
# map 唯一必用场景就是配合 input().split()
```

## 💬 德远的笔记

> `round(x, 2)` 和 `f"{x:.2f}"` 这个区别太重要了。OJ 判输出格式时，`round(7, 2)` 输出 `7` 而不是 `7.00`，直接 WA。以后只要 OJ 要求「保留 X 位小数」，无脑用 f-string：`f"{ans:.2f}"`。
>
> 还有 `max(count, key=count.get)` 这个当时觉得像天书——其实就是「按 count 的值来比较，返回最大的那个 key」。理解了 `key=` 的含义之后，`max(words, key=len)` 之类的都一通百通。
