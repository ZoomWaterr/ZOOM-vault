# 高频陷阱

## 先看这个

这里装的是“我明明会，但经常因为细节摔一下”的东西。

## 列表坑

```python
bad = [[0] * 5] * 3
good = [[0] * 5 for _ in range(3)]
```

- `list.pop(0)` 是 O(n)，队列用 `deque`。
- 循环里频繁 `x in list` 可能 TLE，先转 `set`。
- 遍历时删除元素容易漏，优先新建列表。

## 字符串坑

```python
s.strip()      # s 没变
s = s.strip()  # 这样才变
```

## 字典坑

```python
count[x] = count.get(x, 0) + 1
```

不要一上来 `count[x] += 1`。

## 循环坑

嵌套循环里 `i` 经常只是“轮数”，不一定是当前位置。变量含义复杂时直接用中文变量名。

```python
for 轮数 in range(n):
    for 当前位置 in range(n - 1 - 轮数):
        if nums[当前位置] > nums[当前位置 + 1]:
            nums[当前位置], nums[当前位置 + 1] = nums[当前位置 + 1], nums[当前位置]
```

## 数学坑

- `/` 返回 float，`//` 才是整数除法。
- `int()` 是截断，不是四舍五入。
- 浮点数比较用误差。
