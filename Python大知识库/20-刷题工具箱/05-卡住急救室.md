# 🚑 卡住急救室

> 卡住不要先骂自己。先判断卡在哪一层。

---

## A. 我不知道题目要我干嘛

先写一句人话：

```text
题目给我：____
要我求：____
我每一步要维护：____
最后输出：____
```

如果这四格填不出来，就别急着写代码。

## C. 数学思路不清楚

你代码里出现过“使用数学手法写得自己思维不清楚，不如下面的直接暴力”。这不是废话，这是一种很真实的卡点。

处理方式：

1. 写 `n = 1, 2, 3, 4, 5` 的答案。
2. 别急着公式，先看变化。
3. 猜规律。
4. 用 `n = 6` 验证。
5. 再写代码。

**模糊感不是错误，它只是还没被小样例照亮。**

## D. 暴力能过，但感觉不优

暴力是第一稿，不丢人。然后问一句：

| 我重复做了什么 | 可能的优化 |
|:---|:---|
| 反复求一段和 | 前缀和 |
| 区间里每个数都改一遍 | 差分 |
| 反复查某个东西出现过没有 | 集合/字典 |
| 反复算前面的状态 | 递推/DP |
| 在有序东西里找答案 | 二分/双指针 |

---

## E. 样例过了，但我不放心

跑三类边界：

- 最小：`n = 1`、一个字符、一个数。
- 极端：全相同、全不同、全 0、全负数。
- 格式：空格、换行、小数位、有没有列表括号。

---

## F. 我完全烦了

那就写最小可运行骨架：

```python
def solve():
    # 读输入
    # 处理
    # 输出
    pass

solve()
```

先让文件不空，脑子就会开始接着走。
