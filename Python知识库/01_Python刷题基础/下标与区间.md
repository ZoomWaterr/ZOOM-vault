# 下标与区间

下标和区间是算法题里最容易让人脑子糊的地方。先把规则固定下来。

## Python 下标

```python
nums = [10, 20, 30]
nums[0]  # 10
nums[1]  # 20
nums[2]  # 30
```

长度是 `n`，合法下标是 `0 ~ n-1`。

## 左闭右开

Python 切片是左闭右开：

```python
nums[l:r]
```

包含 `l`，不包含 `r`。

## 为什么前缀和常用 n+1

```python
prefix[0] = 0
prefix[i + 1] = prefix[i] + nums[i]
```

这样 `[l, r]` 的和就是：

```python
prefix[r + 1] - prefix[l]
```

你不用特判 `l = 0`。

## 区间写法对比

| 写法 | 含义 |
|---|---|
| `[l, r]` | 包含左右端点 |
| `[l, r)` | 包含左，不包含右 |
| `range(l, r)` | 也是 `[l, r)` |
| `nums[l:r]` | 也是 `[l, r)` |

## 常见坑

### 坑 1：`range(n)` 不到 n

```python
for i in range(n):
    ...
```

最后一个 `i` 是 `n - 1`。

### 坑 2：`r + 1` 越界

差分里常写：

```python
diff[r + 1] -= c
```

所以数组要多开：

```python
diff = [0] * (n + 2)
```

### 坑 3：题目 1 下标，代码 0 下标

如果题目给的是第 1 个到第 n 个，你要决定：

- 直接用 1 下标数组，少转换。
- 或者全部转成 0 下标，公式统一。

不要一半 1 下标，一半 0 下标。
