# 解题全流程 · 从读题到 AC

## 五步框架

```
STEP 1 读懂题意
├─ 大白话复述一遍：这题到底要我干嘛？
├─ 圈关键数字：n最大多少？有负数？要取模？
└─ 样例先手算一遍

STEP 2 建立模型
├─ 判断题型：求最值？计数？判断可行？
└─ 抽象成：输入 → 处理 → 输出

STEP 3 匹配算法 ← 翻 [[01-题型模式识别]]
├─ "连续子数组" → 滑动窗口/前缀和
├─ "最短路径" → BFS
├─ 拿不准 → 看n范围倒推

STEP 4 写代码
├─ 搭框架：def solve() → 输入 → 核心逻辑 → 输出
├─ 核心逻辑翻 [[../30-算法模板/00-算法总览|算法模板]]
└─ 写完主逻辑再加边界

STEP 5 验证
├─ 跑样例 → 过了再提交
├─ 没过 → [[02-边界检查与调试]]
```

---

## 输入规模 → 复杂度

| n | 可用复杂度 | 算法 |
|:---:|:---|:---|
| ≤10 | O(n!) | 全排列 |
| ≤20 | O(2ⁿ) | 状态压缩 |
| ≤100 | O(n³) | Floyd, 区间DP |
| ≤2000 | O(n²) | 双重循环, 冒泡 |
| ≤10⁵ | O(n log n) | 排序, 二分, 堆 |
| ≤10⁶ | O(n) | 前缀和, 双指针 |

Python ≈ 10⁷次/秒

## 赛中策略

```
1. 通读所有题(10分钟)→标简单/中等/难
2. 先秒最简单的
3. 按"会做"优先，不按题号
4. 提交前30秒跑边界检查
5. 卡20分钟→跳过
6. 最后20分钟复查，不写新题
```
