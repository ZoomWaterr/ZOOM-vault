**快速定位**
先写 `solve()`，核心逻辑再单独抽函数。

**一句话**
函数不是装正式，是把一团代码拆成能调试的块。

**适用场景**
- 代码超过 20 行
- 逻辑复用
- bug 找不到位置

**原理理解**
输入、核心逻辑、输出分开，大脑一次只处理一件事。

**标准模板**

```python
def 核心逻辑(nums):
    return sum(nums)

def solve():
    nums = list(map(int, input().split()))
    print(核心逻辑(nums))

if __name__ == "__main__":
    solve()

```

**易错点**
- 刷题也可以写函数
- 函数先定义再调用
- return 和 print 不要混乱

`stats.py` 证明你会写结构；现在要把这个能力迁移到刷题。
