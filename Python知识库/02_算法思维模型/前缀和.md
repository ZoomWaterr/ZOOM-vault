# 前缀和

## 一句话理解

前缀和就是提前把“从开头到当前位置的和”存起来，让区间和可以 `O(1)` 算。

## 题目信号

- 多次问 `[l, r]` 的和。
- 连续一段的总和。
- 子数组和。
- 二维区域和。

## 暴力做法

每次查询都从 `l` 加到 `r`：

```python
s = 0
for i in range(l, r + 1):
    s += nums[i]
```

单次 `O(n)`，查询多了会慢。

## 优化思想

`prefix[i]` 表示前 `i` 个数的和。

```python
prefix[0] = 0
prefix[i + 1] = prefix[i] + nums[i]
```

那么：

```python
sum(l, r) = prefix[r + 1] - prefix[l]
```

## 模板

```python
nums = list(map(int, input().split()))
n = len(nums)

prefix = [0] * (n + 1)
for i in range(n):
    prefix[i + 1] = prefix[i] + nums[i]

# 查询 [l, r]
ans = prefix[r + 1] - prefix[l]
```

## 复杂度

- 预处理：`O(n)`。
- 每次查询：`O(1)`。
- 空间：`O(n)`。

## 易错点

- `prefix` 通常开 `n + 1`，这样区间公式更统一。
- 注意题目下标是从 0 还是从 1 开始。
- 前缀和适合“静态数组多次查询”，如果中间经常修改，要考虑别的方法。
