# 从暴力到优化

## 先看这个

先想暴力不是丢人。暴力是拿来理解题目的，优化是拿来通过数据范围的。

## 三步走

1. 写出最直观做法。
2. 估算复杂度。
3. 找重复计算在哪里。

## 常见优化方向

| 暴力问题 | 常见优化 |
|:---|:---|
| 每次求区间和 | 前缀和 |
| 每次改一段区间 | 差分 |
| 双重循环找两个数 | 排序 + 双指针 / 字典 |
| 线性查找很多次 | set / dict / 二分 |
| 递归重复算 | 记忆化 / DP |

## 别想复杂

优化不是把代码写玄。优化通常是把“重复做的事”提前存起来。
