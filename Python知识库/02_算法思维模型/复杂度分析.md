# 复杂度分析

复杂度回答一个问题：数据变大时，程序会不会跑得动。

## 常见复杂度

| 复杂度 | 感觉 |
|---|---|
| `O(1)` | 一步完成 |
| `O(log n)` | 每次砍一半 |
| `O(n)` | 扫一遍 |
| `O(n log n)` | 排序常见 |
| `O(n^2)` | 双层循环 |
| `O(2^n)` | 枚举所有选择 |
| `O(n!)` | 全排列 |

## 怎么估

单层循环：

```python
for i in range(n):
    ...
```

是 `O(n)`。

双层循环：

```python
for i in range(n):
    for j in range(n):
        ...
```

是 `O(n^2)`。

每次除以 2：

```python
while n > 1:
    n //= 2
```

是 `O(log n)`。

## 做题时怎么用

先看数据范围：

- `n <= 100`：`O(n^2)` 通常可以。
- `n <= 10^5`：通常要 `O(n)` 或 `O(n log n)`。
- `n <= 20`：可能是状态压缩、回溯、二进制枚举。

## 核心意识

正确但超时，说明你还没真正解决题目。要问：

> 哪些计算重复了？能不能提前存、排序、用哈希、用前缀和、用二分？
