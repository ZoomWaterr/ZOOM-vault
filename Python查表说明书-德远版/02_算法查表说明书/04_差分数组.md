# 差分数组

tags: #算法/差分 #区间

## 应该怎么做

看到“很多次给一个区间整体加/减”，先想差分。

## 这在干什么

你的原话：

> 差分数组记录的就是在区间内的变化量，前缀和把变化量积累成真实值。

核心口诀：

```text
在 L 位置 +C    表示从 L 开始，每人多 C 个
在 R+1 位置 -C  表示到 R 结束，R+1 之后不多了
```

## 优秀模板

```python
n, m = map(int, input().split())
diff = [0] * (n + 1)

for _ in range(m):
    l, r, c = map(int, input().split())
    diff[l] += c
    if r + 1 <= n:
        diff[r + 1] -= c

for i in range(1, n):
    diff[i] += diff[i - 1]

print(*diff[:n])
```

## 什么时候翻它

- “区间加法”
- “批量更新”
- “很多次操作后输出最终数组”
- “一个范围内，批量做同一件事”

## 我的提醒

区间操作就想差分。与其真的把 L 到 R 每个位置都改一遍，不如只打两个标记，最后统一结算。
