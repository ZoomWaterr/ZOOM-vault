## 按 n 的范围倒推（题目问法模糊时的救命稻草）

> 题目词可能模糊，但数据范围不会骗人。

| n 最大多少 | 你能用的算法 | 绝对不能用的 |
|:---:|:---|:---|
| ≤ 10 | 全排列、暴力枚举、O(n!) | — |
| ≤ 20 | 状态压缩 DP、子集生成 O(2ⁿ) | — |
| ≤ 100 | 三重循环 O(n³)、Floyd、区间 DP | O(2ⁿ) |
| ≤ 500 | O(n³) 勉强能过 | O(2ⁿ) |
| ≤ 2,000 | 双重循环 O(n²)、冒泡 | O(n³) 危险 |
| ≤ 10⁵ | 排序 O(n log n)、堆、二分 | O(n²) 必超时 |
| ≤ 10⁶ | 单次遍历 O(n)、前缀和 | O(n log n) 可能卡 |
| ≥ 10⁷ | O(n) 或更优，一次扫描 | 多一层循环都危险 |

## 💬 德远的笔记

> 这张表是整个知识库最有价值的东西。刚开始刷题的时候，拿到一道题最大的困难不是「不会写」，而是「不知道用什么方法写」。后来发现题目里其实到处是暗示——看到「连续子数组」就应该条件反射想到前缀和/滑动窗口，看到「最短路径」就想到 BFS。
>
> 如果关键词实在模糊，那就看 n 的范围。n=100 可以写 O(n³) 三重循环慢慢来，n=10⁵ 必须是 O(n log n) 以下。**数据范围不会骗人。**
