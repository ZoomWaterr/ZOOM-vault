# 🧭 我现在卡在哪

> 卡住时先判断自己卡在第几层，不要直接骂自己不会。

## 第一层：题目到底要我干嘛

先把题目翻译成一句人话：

| 题目问法 | 翻译 |
|:---|:---|
| 求最大/最小 | 我要维护一个最优值 |
| 统计次数 | 我要建一个计数字典 |
| 连续一段 | 可能是前缀和、滑动窗口、双指针 |
| 多次区间加 | 想差分 |
| 每一步依赖前一步 | 想递推或 DP |

## 第三层：我有数学感觉，但不敢写

你的真实代码里出现过这种情况：推了一点，又觉得“思维不清楚”，于是退回暴力。

处理办法很简单：

1. 写 `n = 1, 2, 3, 4, 5` 的答案。
2. 找规律，不急着上代码。
3. 用 `n = 6` 验证规律。
4. 再写代码。

## 第四层：代码写出来了，但怕错

跑这三个：

- 最小值：`n = 1`、一个字符、一个数。
- 极端值：全相同、全不同、全 0、全负数。
- 格式：有没有多空格、列表括号、小数位。
