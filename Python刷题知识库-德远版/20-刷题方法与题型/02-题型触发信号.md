# 题型触发信号

## 先看这个

题型识别不是猜高级算法，而是看到题目关键词后，先想到一批候选工具。

## 按题目在问什么查

| 题目说 | 先想 |
|:---|:---|
| 出现次数、最多、频率 | 字典计数 / Counter |
| 连续一段、区间和 | 前缀和 / 滑动窗口 |
| 多次区间加减 | 差分 |
| 排序后更好处理 | 排序 / 双指针 / 二分 |
| 最小的最大、最大的最小 | 二分答案 |
| 从一个状态走到另一个状态 | BFS |
| 所有方案、排列组合 | DFS / 回溯 |
| 当前最优选择看起来能推出整体最优 | 贪心 |
| 前面结果影响后面 | DP / 递推 |

## 按数据范围查

| n 大小 | 常见可接受做法 |
|:---|:---|
| `n <= 100` | 三重循环可能都能试 |
| `n <= 10^3` | O(n²) 通常可试 |
| `n <= 10^5` | O(n log n) 或 O(n) |
| `n >= 10^6` | 尽量 O(n)，输入输出也要小心 |

## 我容易摔在哪里

你现在还在基础题期，不需要每题都强行套算法。先用触发信号把题目归类，知道自己在干什么。
