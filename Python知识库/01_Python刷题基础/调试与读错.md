# 调试与读错

刷题不是一遍写对才厉害。真正厉害是错了能定位。

## 读错题时先问

- 输入有几行？
- 每行几个数？
- 输出要空格还是换行？
- 下标从 0 还是从 1？
- 区间端点包不包含？
- 数据范围多大？

## 调试变量

当你不知道循环在干嘛时，打印关键变量：

```python
for i in range(n):
    print("i =", i)
    for j in range(i):
        print("  j =", j, "nums[j] =", nums[j], "nums[i] =", nums[i])
```

看清楚以后再删掉调试输出。

## 最小样例调试

不要一上来用大样例。自己造小数据：

```text
3
4 1 3
```

手算一遍，再看程序每一步是否一致。

## 常见错误定位

| 现象 | 优先检查 |
|---|---|
| 少一个答案 | 循环边界 |
| 多一个答案 | 区间是否包含右端点 |
| 样例对，大数据错 | 复杂度或隐藏边界 |
| 报 KeyError | 字典 key 不存在 |
| IndexError | 下标越界 |
| 超时 | 复杂度过高 |

## 一句话

调试不是乱 print，而是打印“当前变量角色是否符合你的想法”。
