# 变量角色

你说写双层 `for` 会迷，本质就是变量角色还不够稳定。

## 变量不是字母，是角色

| 变量 | 常见角色 |
|---|---|
| `i` | 当前正在处理的位置 |
| `j` | 和 `i` 配合比较的另一个位置 |
| `cnt` | 当前这一轮的数量 |
| `ans` | 总答案 |
| `left` | 当前窗口左边界 |
| `right` | 当前窗口右边界 |
| `cur` | 当前状态 |
| `prev` | 前一个状态 |
| `visited` | 是否已经访问过 |

## 写循环前先填这张表

```text
i 表示：
j 表示：
循环范围表示：
每一轮要更新：
最终答案是：
```

## 例子：左边有多少个数比当前小

```python
for i in range(n):
    cnt = 0
    for j in range(i):
        if nums[j] < nums[i]:
            cnt += 1
    ans.append(cnt)
```

角色表：

```text
i 表示：当前数的位置
j 表示：当前数左边的位置
range(i) 表示：只枚举左边，不包含自己
cnt 表示：当前数左边比它小的数量
ans 表示：每个位置的答案列表
```

## 例子：矩阵遍历

```python
for i in range(n):
    for j in range(m):
        print(grid[i][j])
```

角色表：

```text
i 表示：第几行
j 表示：第几列
grid[i][j] 表示：第 i 行第 j 列的元素
```

## 什么时候说明你变量乱了

- 写着写着不知道 `i` 是下标还是值。
- `cnt` 有时表示当前数量，有时表示总数量。
- `ans` 一会儿是列表，一会儿是数字。
- 内层循环也用 `i`。

解决方法很简单：写代码前先写变量角色。
