# 差分

## 一句话理解

差分记录变化量，前缀和把变化量还原成真实值。

## 题目信号

- 多次区间加法。
- `[L, R]` 每个位置都加 `C`。
- 最后才问整个数组结果。

## 暴力做法

每次操作都把 `L` 到 `R` 全部加一遍：

```python
for i in range(L, R + 1):
    a[i] += C
```

一次操作 `O(n)`，`m` 次就是 `O(nm)`。

## 优化思想

不真的把区间填满，只打两个标记：

```python
diff[L] += C
diff[R + 1] -= C
```

最后对 `diff` 做前缀和，就得到真实数组。

## 模板

```python
diff = [0] * (n + 2)

for _ in range(m):
    L, R, C = map(int, input().split())
    diff[L] += C
    diff[R + 1] -= C

for i in range(1, n + 1):
    diff[i] += diff[i - 1]

print(*diff[1:n + 1])
```

## 复杂度

- 每次区间操作：`O(1)`。
- 最后还原：`O(n)`。
- 总复杂度：`O(n + m)`。

## 易错点

- 数组要多开，防止 `R + 1` 越界。
- 差分常用 1 下标，和题目保持一致更省事。
- 如果每次操作后立刻要查结果，普通差分不一定适合。
