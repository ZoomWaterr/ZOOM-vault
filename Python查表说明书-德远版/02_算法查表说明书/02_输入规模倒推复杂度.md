# 输入规模倒推复杂度

tags: #算法/复杂度 #刷题/判断

## 应该怎么做

看 `n` 最大多少，先排除绝对会超时的写法。

| n 最大值 | 大概能用 | 避开 |
|---:|---|---|
| 10 | 全排列、暴力、DFS | 基本都能试 |
| 20 | 状态压缩、子集枚举 | 太多嵌套 |
| 100 | 三重循环 | 指数级 |
| 2,000 | 双重循环 | 三重循环危险 |
| 100,000 | `O(n log n)`、`O(n)` | `O(n^2)` |
| 1,000,000 | `O(n)` | 排序可能看时限 |
| 10,000,000 | 一次扫描或公式 | 多一层循环都危险 |

## 这在干什么

复杂度不是装饰，它决定你是该暴力、优化，还是必须推公式。

## 我的提醒

“数学思路不清楚”时，不要直接放弃。先手算小例子：`n=1,2,3,4,5`，找规律，再验证 `n=6`。
