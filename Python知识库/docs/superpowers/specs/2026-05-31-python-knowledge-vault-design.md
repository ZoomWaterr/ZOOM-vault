# Python Knowledge Vault Design

## Goal

Build an Obsidian-style Markdown knowledge base at `F:\ZOOM-vault\Python知识库` for Python competitive-programming study.

## Audience

The learner already knows basic Python terms and has solved beginner problems, but needs a larger algorithm knowledge network, clearer problem-model recognition, and practical review habits.

## Content Principles

- Use the learner's existing style: 一句话理解、口诀、模板、做过、易错点.
- Avoid over-splitting. For example, 双层 for belongs inside `for循环.md`, not as a separate top-level note.
- Prefer problem-solving models over encyclopedia-style definitions.
- Always connect Python syntax to contest use.
- Use external resources as references, not copied content.

## Structure

- `00_总览`: maps, route, review method, recognition table.
- `01_Python刷题基础`: Python features needed for problem solving.
- `02_算法思维模型`: complexity, enumeration, hash, prefix sum, diff, pointers, binary search.
- `03_数据结构`: arrays, linked lists, stack, queue, hash table, tree, heap, graph.
- `04_进阶算法`: DFS, BFS, backtracking, divide and conquer, DP, greedy, shortest path.
- `05_比赛题型`: practice indexes and training plan.
- `06_错题与复盘`: review templates and failure categories.
- `90_模板库`: copyable Python templates.
- `99_资料来源`: source indexes and usage boundaries.

## First Version Scope

Create the folder structure, core notes, source indexes, and generated practice-code indexes for:

- `F:\Code\PythonCode\洛谷`
- `F:\Code\PythonCode\C语言网`
- `F:\Code\PythonCode\蓝桥云课`

## Non-goals

- Do not clone or reproduce full external books.
- Do not build a web app.
- Do not reorganize the user's practice-code directories.
