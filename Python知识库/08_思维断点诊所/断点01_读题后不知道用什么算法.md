# 断点01_读题后不知道用什么算法

## 症状

读完题后脑子里出现很多词：DP、哈希、排序、二分、DFS，但不知道该选哪个。

## 根因

你在从“题目文字”直接跳到“算法名”，中间少了“问题结构分析”。

## 正确流程

先不要猜算法，按这个顺序问：

```text
1. 题目要我求什么？判断、计数、最优、构造、查询？
2. 数据是什么结构？数组、字符串、树、图、区间？
3. 数据规模多大？暴力能不能过？
4. 有没有关键词？连续、区间、多次查询、最短、出现次数？
5. 暴力哪里重复？
```

## 快速决策表

| 题目在问 | 常见模型 |
|---|---|
| 出现几次 | [[哈希计数]] |
| 某段连续区间的和 | [[前缀和]] |
| 多次区间修改 | [[差分]] |
| 最长连续一段 | [[滑动窗口]] |
| 有序数组里找边界 | [[二分查找]] |
| 最短步数 | [[BFS]] |
| 所有方案 | [[回溯]] |
| 当前结果依赖前面 | [[动态规划]] |
| 每步选当前最好 | [[贪心]] |

## 训练动作

每道题开头写 4 行：

```text
题目类型：
数据结构：
暴力做法：
可能模型：
```

你只要坚持写，题型识别会明显变强。
