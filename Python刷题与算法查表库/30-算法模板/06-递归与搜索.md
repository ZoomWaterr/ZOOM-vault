# 递归与搜索

**快速结论**
递归把问题交给下一层；搜索从当前状态扩展所有可能状态。

**什么时候用**
- 树形结构。
- 当前状态可以走向多个下一状态。
- 需要遍历所有可达状态。
- 问“能不能到达”“有多少连通块”。

**递归三要素**
1. 函数含义：这个函数负责解决什么问题？
2. 终止条件：什么时候不再继续？
3. 递推关系：怎么把问题推进到下一层？

**DFS 模板**
```python
def dfs(state):
    if is_end(state):
        return

    for nxt in next_states(state):
        if nxt in seen:
            continue
        seen.add(nxt)
        dfs(nxt)
```

**网格 DFS 模板**
```python
dirs = [(1,0), (-1,0), (0,1), (0,-1)]

def dfs(r, c):
    if r < 0 or r >= n or c < 0 or c >= m:
        return
    if grid[r][c] == '#' or (r, c) in seen:
        return

    seen.add((r, c))
    for dr, dc in dirs:
        dfs(r + dr, c + dc)
```

**常见变体**
- 连通块数量。
- 岛屿数量。
- 树的遍历。
- 记忆化搜索：DFS + 缓存。

**复杂度**
通常是 `O(状态数 + 转移数)`。

**易错点**
- 忘记终止条件会无限递归。
- 忘记 visited 会重复访问。
- Python 递归深度有限，深递归要小心。
