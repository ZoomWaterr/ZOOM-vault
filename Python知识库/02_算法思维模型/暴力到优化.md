# 暴力到优化

算法学习真正的核心是：从暴力出发，找到为什么慢，再优化。

## 暴力不是坏事

暴力法有三个作用：

1. 帮你确认题意。
2. 帮你得到一个正确但可能慢的方案。
3. 暴露重复计算在哪里。

## 优化问题的固定问法

```text
1. 我现在有几层循环？
2. 每层循环在枚举什么？
3. 哪些东西被重复算了？
4. 能不能提前存起来？
5. 能不能利用有序性？
6. 能不能把一段的计算变成 O(1)？
```

## 常见优化路线

| 暴力慢在哪里 | 优化方向 |
|---|---|
| 每次都查某个数出现没 | [[dict字典与哈希]] / [[set集合]] |
| 每次都重新统计次数 | [[哈希计数]] |
| 每次都重新算区间和 | [[前缀和]] |
| 每次都给一整段加值 | [[差分]] |
| 双层循环找有序数组中的两数 | [[双指针]] |
| 一个个试答案 | [[二分答案]] |
| 递归里重复算同一状态 | [[动态规划]] |

## 例子：区间和

暴力：

```python
total = 0
for i in range(l, r + 1):
    total += nums[i]
```

如果查询很多次，这段会重复扫区间。

优化：

```python
prefix[r + 1] - prefix[l]
```

这就是用空间换时间。

## 例子：统计出现次数

暴力：

```python
for x in nums:
    cnt = nums.count(x)
```

`count` 每次都扫一遍，很慢。

优化：

```python
count = {}
for x in nums:
    count[x] = count.get(x, 0) + 1
```

扫一遍就够。

## 一句话

不要只问“该用什么算法”，先问“暴力重复做了什么”。
