# 流程控制 · if / for / while

> 程序的骨架。条件判断、循环遍历、打断跳出——这些用好了代码就不会乱。

---

## if 条件判断

```python
if 条件1:
    操作1
elif 条件2:          # 可以有多个 elif
    操作2
else:                # else 不需要条件
    操作3
```

### 判断的"真"与"假"

这些值会被当成 **False**：`0`、`0.0`、`""`（空字符串）、`[]`（空列表）、`{}`（空字典）、`set()`（空集合）、`None`

**其余全是 True**。所以可以直接写：

```python
if 列表:       # 列表非空 → True
if 字符串:     # 字符串非空 → True
```

**连续比较**——Python 独有：

```python
最低分 <= 我的分数 <= 最高分    # 等价于 (>=最低分) and (<=最高分)
```

## for 循环 — 遍历

```python
# 跑 n 次
for i in range(5):              # i = 0, 1, 2, 3, 4
for i in range(2, 8):           # i = 2, 3, 4, 5, 6, 7
for i in range(0, 10, 2):       # i = 0, 2, 4, 6, 8

# 遍历列表
for 元素 in 列表:
    print(元素)

# 同时拿序号和值
for idx, val in enumerate(列表):
    print(f"第{idx}个是{val}")

# 同时遍历两个列表
for a, b in zip(列表1, 列表2):
    print(a, b)
```

### range() → 生成数字序列

```python
range(5)         # → 0, 1, 2, 3, 4
range(2, 8)      # → 2, 3, 4, 5, 6, 7
range(0, 10, 2)  # → 0, 2, 4, 6, 8   步长为 2
```

## while 循环 — 条件满足就继续

```python
while 条件:
    循环体
```

**for vs while 怎么选**：

| | for | while |
|:---|:---|:---|
| 适用 | 遍历数据集、已知次数 | 条件满足就一直循环 |
| 关注 | 关注每个元素 | 关注循环条件 |

## break 与 continue

```python
for x in 列表:
    if x < 0:
        continue        # 跳过当前，进入下一轮
    if x == 0:
        break           # 彻底退出循环
    print(x)            # 只打印正数，遇 0 就停
```

- `break`：跳出整个循环（while 后的 else 也不会执行）
- `continue`：跳过本轮剩余代码，直接下一轮

## for-else — Python 独有特性

```python
for num in nums:
    if num == target:
        print("找到了")
        break
else:               # ← for 正常跑完（没 break）才执行
    print("没找到")
```

> 非常好用！比如在「2973 出现次数超过一半的数」里，遍历完没找到超过一半的就输出 "no"。

## 双重循环 — 不重不漏找数对

```python
for i in range(n):          # 外层：逐个扫描每个数
    for j in range(i):      # 内层：让 i 和它左边所有数比较
        # 这里 i 和 j 就是所有 j < i 的组合
```

## ⚠️ 嵌套循环易错点

> 这是最容易出 bug 的地方。**给循环变量起有意义的名字**，别用 i、j。

```python
# ❌ 容易混
for i in range(n):
    for j in range(n-1-i):
        if nums[j] > nums[i]:      # ← 应该是 j+1，不是 i！

# ✅ 清晰
for 轮数 in range(n):
    for 当前位置 in range(n - 1 - 轮数):
        if nums[当前位置] > nums[当前位置 + 1]:
```

## 💬 德远的笔记

> 嵌套循环用 i、j 是跟 C/C++ 教材学的习惯，但 Python 里这种模糊命名真的坑死人。在「车厢重组 (2117)」里，我把比较 `j` 和 `j+1` 写成了比较 `j` 和 `i`，结果整个冒泡排序的逻辑都乱了。从那以后，只要循环套循环，外层叫「轮数」「行」，内层叫「当前位置」「列」——名字自己会说话，犯错的概率小多了。
>
> 还有 for-else 真的是 Python 的好设计。判断「有没有找到」不用额外设一个 flag 变量。
