# 02 错因地图

错因不是为了贴标签，而是为了下次少花时间。

---

## 语法层

| 错因 | 对应章节 |
|:---|:---|
| 输入没转类型 | [输入、输出与数据流](../10-Python语言基础/02-输入、输出与数据流.md) |
| 列表输出成 `[1, 2, 3]` | [输入、输出与数据流](../10-Python语言基础/02-输入、输出与数据流.md) |
| 字符串方法没接回来 | [字符串](../20-数据组织/02-字符串.md) |
| 嵌套循环变量混 | [循环与迭代](../10-Python语言基础/05-循环与迭代.md) |

## 思路层

| 错因 | 处理方式 |
|:---|:---|
| 数学推导模糊 | 先写 1、2、3、4、5 的小样例 |
| 暴力能过但不优 | 看有没有重复劳动 |
| 不知道维护什么变量 | 回到“输入、维护、输出”三问 |

## 习惯层

| 错因 | 处理方式 |
|:---|:---|
| 样例过了就提交 | 跑最小、极端、格式三类边界 |
| 代码全平铺 | 先尝试 `solve()` |
| 注释残留太乱 | 保留有思考价值的注释，清理无意义尝试 |
