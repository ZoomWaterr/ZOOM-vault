# 🧭 解题全流程 · 从读题到 AC

> 这页不是比赛玄学，是你卡住时的路线图。

---

## 第 0 步：别急着写代码

先问一句：**这题到底要我输出什么？**

很多错不是算法错，而是题目目标读歪了。

---

## 第 1 步：把题目翻译成人话

```text
输入：____
过程：____
输出：____
```

例子：

```text
输入：一堆数字
过程：统计每个数字出现几次
输出：有没有某个数字出现超过一半
```

这时候方法就已经浮出来了：字典计数。

---

## 第 2 步：看题型信号

| 看到 | 想 |
|:---|:---|
| 出现次数、最多、超过一半 | 字典计数 |
| 连续一段 | 当前长度 / 前缀和 / 双指针 |
| 区间批量修改 | 差分 |
| 第 n 项、楼梯、斐波那契 | 递推 |
| 排序后更好处理 | sort / sorted |

---

## 第 3 步：先写骨架

```python
def solve():
    # 1. 读输入
    # 2. 核心逻辑
    # 3. 输出
    pass

solve()
```

你不一定每题都要函数化到很漂亮，但这个骨架能让脑子分区。

---

## 第 4 步：写核心逻辑

只盯一个变量：**答案到底怎么更新？**

- 计数题：`count[x] += 1`
- 最值题：`best = max(best, cur)`
- 模拟题：按题目规则一步步更新
- 递推题：`dp[i] = ...`

---

## 第 5 步：提交前 30 秒

- `n = 1` 会怎样？
- 全相同会怎样？
- 输出有没有多空格、列表括号、小数位问题？

---
