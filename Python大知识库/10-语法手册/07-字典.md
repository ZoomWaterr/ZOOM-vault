# 字典 dict

> 在其他语言里叫哈希表 (hash) / 映射 (map)。刷题里**统计、映射、存关系**全靠它。

---

## 一句话理解

一本名册——一个名字 (key) 对应一个值 (value)。通过名字能瞬间找到对应的值。

---

## 创建

```python
d = {}                          # 空字典
d = {"张三": 95, "李四": 88}    # 直接写
d = dict()                      # 另一种空字典
```

---

## 增 / 改 / 查 / 删

```python
# 增或改 — key 不存在就加，存在就覆盖
d["王五"] = 76

# 查 — 两种方式
d["张三"]               # 直接取，key 不存在 → KeyError 报错！
d.get("赵六", 0)        # 安全取，key 不存在 → 返回默认值 0
#              ↑ 找不到时返回这个

# 删除
del d["张三"]
```

---

## 频率统计模板 — 必背

```python
count = {}
for x in 数据:
    count[x] = count.get(x, 0) + 1
```

这就是传说中的「字典计数」。`get(x, 0)` 意思是：有 x 就返回它的次数，没有就返回 0。然后 `+1`。

---

## 找出现最多的

```python
best = max(count, key=count.get)
#               ↑ 关键：key=字典.get 意思是「按 value 来比大小」
print(best, count[best])
```

---

## 判断是否超过一半（附赠 for-else 用法）

```python
for num, cnt in count.items():
    if cnt > n / 2:
        print(num)
        break
else:               # ← 这个 else 和 for 配对！
    print("no")     # for 正常跑完（没 break）才执行
```

---

## 遍历

```python
# 只遍历 key
for k in d:             # 等价于 for k in d.keys()
    print(k)

# 遍历 key + value ← 最常用！
for k, v in d.items():
    print(f"{k} → {v}")
```

---

## 什么时候用字典

| 场景 | 说明 |
|:---|:---|
| "统计出现次数" | `count[x] = count.get(x, 0) + 1` |
| "记录每个 x 对应的 y" | 如：名字→分数、数字→下标 |
| "找出最多/最少的" | `max(count, key=count.get)` |
| "判断某个东西出现过没有" | `if x in dict:` — O(1)！ |

---

## 时间复杂度

所有操作几乎全是 **O(1)**：查、增、改、删、判存在。

```
d[key]         → O(1)
d.get(key)     → O(1)
d[key] = val   → O(1)
del d[key]     → O(1)
key in d       → O(1)
```

> 🔥 这就是为什么字典比列表快——列表查元素是 O(n)，字典是 O(1)。当 n 很大时天差地别。

---

## 进阶：defaultdict 和 Counter

```python
from collections import defaultdict, Counter

# defaultdict：不用每次都 get，自动给不存在的 key 赋默认值
count = defaultdict(int)        # 默认值是 0
for x in 数据:
    count[x] += 1               # 不用判断 key 存不存在！

# Counter：一行搞定统计
count = Counter(数据)
count.most_common(3)            # 出现最多的前 3 个
```

## 💬 德远的笔记

> `count.get(key, 0)` 这个手法一开始总觉得别扭，后来在「出现次数超过一半的数 (2973)」那道题里反复用了好几次才真正掌握。思路很简单：遍历数组，每个数都在字典里记一次数，然后遍历字典找有没有超过一半的。
>
> 关键就是**安全取出**——用 get 而不是直接 `count[x]`。你要直接取，第一次碰到这个 key 时，字典里还没有，直接 KeyError 程序崩了。
>
> 还有 `max(count, key=count.get)` 这行，第一次看到觉得是天书——怎么 `max` 还能传 `key` 参数？后来理解了：`key=xxx` 就是告诉 max「按什么标准比大小」。`count.get` 就是「取这个 key 对应的 value」。所以这句话翻译过来就是：**找 value 最大的那个 key**。神来之笔！
