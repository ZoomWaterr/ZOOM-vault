# 题型识别总表

刷题时先看题目“信号”，再决定用什么方法。这个表就是把题目语言翻译成算法模型。

| 题目信号 | 优先想到 | 典型变量 |
|---|---|---|
| 统计某个数/字符出现几次 | [[哈希计数]] | `count[x]` |
| 判断某元素是否出现过 | [[set集合]] / [[dict字典与哈希]] | `seen` |
| 多次查询区间和 | [[前缀和]] | `prefix` |
| 多次给区间加同一个数 | [[差分]] | `diff` |
| 连续子数组、连续子串 | [[滑动窗口]] / [[前缀和]] | `left`, `right` |
| 两边往中间靠 | [[双指针]] | `l`, `r` |
| 有序数组里查找 | [[二分查找]] | `left`, `right`, `mid` |
| 求满足条件的最小最大答案 | [[二分答案]] | `check(x)` |
| 所有选择、所有排列组合 | [[回溯]] | `path`, `used` |
| 从起点走到所有能到的位置 | [[DFS]] / [[BFS]] | `visited` |
| 最短步数、层数扩散 | [[BFS]] | `queue`, `step` |
| 当前答案依赖前面答案 | [[动态规划]] | `dp[i]` |
| 每一步选当前最优 | [[贪心]] | 排序、局部最优 |
| 动态维护最大/最小 | [[堆]] | `heap` |

## 使用方法

做题前不要直接写代码，先写一句：

```text
这题的信号是：____
所以我先考虑：____
暴力做法是：____
暴力复杂度是：____
```

这个动作能把你从“凭感觉写”拉到“按模型思考”。

## 如果你不知道该用什么

1. 我能不能写暴力？见 [[暴力到优化]]。
2. 变量分别代表谁？见 [[变量角色]]。
3. 区间下标有没有搞清？见 [[下标与区间]]。
4. 当前局面是不是一个状态？见 [[状态思想]]。
5. 这题卡住了怎么复盘？见 [[如何复盘一道题]]。
