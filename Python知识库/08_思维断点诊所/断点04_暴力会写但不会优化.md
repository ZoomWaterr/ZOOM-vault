# 断点04_暴力会写但不会优化

## 症状

能写出正确暴力，但一看数据范围就知道会超时，却不知道怎么改。

## 根因

你只看到“代码慢”，没定位“重复计算在哪里”。

## 优化前先问

```text
最慢的是哪一层循环？
内层每次在重复查什么？
这个东西能不能提前存？
能不能利用有序性？
能不能用一个数据结构快速查？
```

## 对应优化

| 暴力重复做的事 | 可能优化 |
|---|---|
| 反复查某元素是否出现 | [[set集合]] |
| 反复统计次数 | [[哈希计数]] |
| 反复计算区间和 | [[前缀和]] |
| 反复修改整个区间 | [[差分]] |
| 双层找有序数组答案 | [[双指针]] / [[二分查找]] |
| 递归重复算同一状态 | 记忆化 / [[动态规划]] |

## 例子

暴力区间和：

```python
for l, r in queries:
    total = 0
    for i in range(l, r + 1):
        total += nums[i]
```

重复点：每次都重新加一段。

优化：[[前缀和]]。
