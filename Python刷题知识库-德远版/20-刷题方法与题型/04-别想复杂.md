# 别想复杂

## 先看这个

你说自己会“想到了，但没法和语法结合”，还会多想。这页专门处理这个问题。

## 简化口诀

- 先用一句话说题目：它到底要我输出什么？
- 先写最普通的变量名：`total`、`count`、`ans`。
- 先写直观循环：能跑通小样例再优化。
- 先让代码变清楚，再让代码变短。

## 三种危险信号

| 信号 | 处理 |
|:---|:---|
| 一上来想高级算法 | 回到暴力和数据范围 |
| 变量越来越多，自己也分不清 | 重命名变量，拆步骤 |
| 写到一半不知道在写啥 | 回到读题四格 |

## 个人提醒

你有一个很好的特质：会在代码里写“这个地方是精华”“神来之笔”“我靠”。这个其实说明你对顿悟很敏感。现在要做的是把顿悟沉淀成稳定套路。
