# 🧭 解题全流程 · 从读题到 AC

> 拿到一道题 → 不知道从哪里下手？按这 5 步走。

---

## 五步框架

```
STEP 1  读懂题意
├─ 用大白话复述一遍：这题到底要我干嘛？
├─ 圈出关键数字：n 最大多少？有负数吗？要取模吗？
└─ 题目给了样例？先手算一遍，确认自己理解了

STEP 2  建立模型
├─ 判断题型：求最值？计数？判断可行？输出方案？
└─ 把题目描述抽象成：输入 → 处理 → 输出

STEP 3  匹配算法  ← 翻 [[01-题型模式识别|题型模式识别表]]
├─ 看到"连续子数组" → 滑动窗口 / 前缀和
├─ 看到"所有方案"   → 回溯 / DFS
├─ 看到"最短路径"   → BFS
└─ 拿不准 → 看 n 的范围倒推（翻下面复杂度对照表）

STEP 4  写代码
├─ 先搭框架：def solve() → 输入 → 核心逻辑 → 输出
├─ 核心逻辑翻 [[../30-算法模板/00-算法总览|算法模板库]]，复制模板改
└─ 写完主逻辑再加边界处理

STEP 5  验证
├─ 跑题目样例 → 过了再提交
├─ 没过？跑 [[03-边界检查清单|边界检查清单]]
└─ 还是没过？→ [[04-调试技巧|调试技巧]]
```

## 完整示例

题目：给一个数组和整数 K，求和为 K 的连续子数组有几个

| 步骤 | 具体思考 |
|:---|:---|
| Step 1 | n ≤ 20000，可能有负数，求**个数**（不是最长/最短） |
| Step 2 | "连续子数组" + "和为 K" → 子数组和 = 前缀和之差。问题变成了：有多少对前缀和之差 = K |
| Step 3 | 「前缀和 + 统计」 → 翻 [[../30-算法模板/01-前缀和与差分|前缀和]] + [[../10-语法手册/07-字典|字典计数]] |
| Step 4 | 遍历一遍，用字典记录每个前缀和出现了几次，同时查 `当前前缀和 - K` 出现次数累加 |
| Step 5 | 试 `[1,1,1] K=2` → 2 对吗？试 `K=0`？试 `n=1`？ |

## 输入规模 → 算法复杂度对照

> Python 约 10⁷ 次简单操作/秒。看 n 选算法——这是**最可靠的决策依据**。

| n 范围 | 可接受复杂度 | 可以用的算法 |
|:---:|:---|:---|
| ≤ 10 | O(n!) | 全排列、暴力回溯 |
| ≤ 20 | O(2ⁿ) | 状态压缩 DP、子集枚举 |
| ≤ 100 | O(n³) | Floyd、区间 DP |
| ≤ 2,000 | O(n²) | 双重循环 DP、冒泡排序 |
| ≤ 10⁵ | O(n log n) | 排序、二分、堆、贪心 |
| ≤ 10⁶ | O(n) | 前缀和、双指针、滑动窗口 |
| ≥ 10⁷ | O(n) 或更小 | 贪心、单次遍历 |

## 赛中策略

```
1. 通读所有题（10 分钟）→ 标上「简单/中等/难」
2. 先秒最简单的那道，建立信心
3. 中档题按"会做"优先，不按题号顺序
4. 每题提交前 30 秒 → 跑边界检查清单
5. 一道题卡 20 分钟 → 果断跳过，回头再看
6. 最后 20 分钟 → 复查已过的代码，不再写新题
```
