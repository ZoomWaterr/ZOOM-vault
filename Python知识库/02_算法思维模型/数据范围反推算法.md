# 数据范围反推算法

比赛题里，数据范围经常在暗示你该用什么复杂度。

## 速查表

| 数据范围 | 大概允许 | 常见算法 |
|---|---|---|
| `n <= 10` | 很高 | 全排列、暴搜 |
| `n <= 20` | `2^n` 左右 | 状态压缩、子集枚举 |
| `n <= 100` | `n^3` 可能可以 | Floyd、区间 DP 入门 |
| `n <= 1000` | `n^2` | 双层循环、基础 DP |
| `n <= 10^5` | `n log n` / `n` | 排序、哈希、前缀和、双指针 |
| `n <= 10^6` | `n` | 线性扫描、前缀、差分 |
| `n` 极大 | `log n` | 二分、数学 |

## 怎么用

题目一读完，先写：

```text
n 最大是：
我暴力复杂度是：
能不能过：
如果不能，慢在哪里：
```

## 例子

如果 `n = 10^5`，你写：

```python
for i in range(n):
    for j in range(n):
        ...
```

这是 `10^10` 级别，基本不可能过。

你就要想：

- 能不能哈希？
- 能不能排序？
- 能不能前缀和？
- 能不能双指针？
- 能不能二分？

## 注意

数据范围不是唯一判断标准，但它能帮你排除明显不可能的做法。
