# 🔧 调试技巧 · Bug 定位三法

> 代码跑不过别慌。三招帮你定位问题。

---

## 方法 1：print 大法（最实用）

```python
def solve():
    data = list(map(int, input().split()))
    print(f"DEBUG 读入数据: {data}")       # 先确认输入对不对

    result = process(data)
    print(f"DEBUG 中间结果: {result}")     # 看中间计算有没有问题

    print(result)                          # 最终输出
```

> 删掉或注释掉 debug print 再提交——OJ 不会容忍多余输出。

## 方法 2：二分定位

把代码一段段注释掉，看哪段注释后 bug 消失，就在那段里。

```
大致思路：
1. 注释掉后半段 → 运行 → 还错？ → bug 在前半段
2. 注释掉前半段 → 运行 → 对了？ → bug 在刚注释的那段
3. 反复缩范围，直到定位到具体那几行
```

## 方法 3：缩到最小用例

> 如果大样例报错但你不知道为啥——把输入缩到 n=2~3，手算出正确答案，和你的输出对比。

```python
# 比如原始输入是 50 个数，你不知道哪个位置出错了
```

## 常见调试信号的含义

```python
# IndexError: list index out of range
→ 数组下标越界。检查所有 range() 范围，尤其是嵌套循环。
→ 数组多开几位：d = [0] * (n + 2) 而不是 d = [0] * n

# KeyError: 'xxx'
→ 字典里没有这个 key。检查是不是用了 d[key] 而不是 d.get(key, default)

# RecursionError: maximum recursion depth exceeded
→ DFS 递归太深。在文件开头加 import sys; sys.setrecursionlimit(1000000)

# TypeError: unsupported operand type(s)
→ 类型不匹配。常见：列表切片减整数、字符串和数字运算

# Time Limit Exceeded (TLE)
→ 看时间复杂度。list.pop(0)、if x in list 在循环里是经典原因
```
