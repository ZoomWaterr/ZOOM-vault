# ✅ 边界检查清单 · 提交前 30 秒

> 样例过了 ≠ 代码对了。提交前花 30 秒过一遍这个清单，能避免 80% 的 WA。

---

## 必查 6 问

| # | 问题 | 具体怎么想 |
|:---:|:---|:---|
| 1 | **n = 1 时**代码对吗？ | 手动跑一遍最小规模，看输出是否合理 |
| 2 | **n = 最大时**会超时/溢出吗？ | 心算复杂度 × 最大 n 的量级 |
| 3 | **全是相同值**还能对吗？ | 试 `[5,5,5]` 或 `"aaaa"` |
| 4 | **负数/零**会出问题吗？ | 试 `[0, -1, -2]`，除法特别注意 |
| 5 | **空输入**（空串、空列表）？ | 试 `""` 或 `[]` |
| 6 | **要取模吗**？ | 如果需要（mod 10⁹+7），加法/减法取模后要 +MOD 防负数 |

## 提交前速查

```
□ 样例输出和题目一模一样？（空格、换行、小数位数）
□ n = 1 手动测过了吗？
□ 变量都初始化了吗？（计数的、累加的）
□ 多组输入时，每组之间清空状态了吗？
□ 输出格式：空格分隔用 print(*a)，不要末尾多空格
□ 整数除法用 // 而不是 / 吗？
```

## 常见错误速查

### 运行时错误 (RE)

| 报错 | 最常见原因 | 怎么修 |
|:---|:---|:---|
| `IndexError: list index out of range` | 数组下标越界 | 检查 range() 范围，数组多开几位 |
| `KeyError` | 字典 key 不存在直接取了 | 换成 `d.get(key, 默认值)` |
| `RecursionError` | DFS 递归太深 | 开头加 `sys.setrecursionlimit(1000000)` |
| `ZeroDivisionError` | 除法分母为 0 | 除法前加 `if b != 0` |
| `ValueError` | 输入有空行或格式不对 | 读入时加 `.strip()` |

### 答案错误 (WA)

| 现象 | 可能原因 | 怎么修 |
|:---|:---|:---|
| 样例对了但交上去 WA | 没处理边界 | 跑这个清单！ |
| 输出和答案差不多 | 用了 `/` 而不是 `//` | 整数运算全部 `//` |
| 原数组莫名其妙被改了 | `sort()` 改了原列表 | 不想改原列表用 `sorted()` |
| 循环里删元素结果少处理了 | 遍历中修改列表 | 倒序遍历或推导式新建 |

### 超时 (TLE)

| 写法 | 为什么要命 | 怎么改 |
|:---|:---|:---|
| `list.pop(0)` | O(n)！ | 换成 `deque.popleft()` |
| `if x in list` 在循环里 | O(n²) | 把 list 转成 set |
| `s += ch` 循环拼接字符串 | 越拼越慢 | 用 list 收集，最后 `"".join()` |
| 数据 10⁵ 写了 O(n²) | 10¹⁰ 次操作 | 看 n 范围改算法 |
