# 边界条件

边界条件是“程序最容易出错的极端情况”。

## 三个基础问题

做完题后至少问：

1. 最小值会怎样？
2. 最大值会怎样？
3. 空或特殊情况会怎样？

## 常见边界

| 类型 | 例子 |
|---|---|
| 最小输入 | `n = 1` |
| 空结构 | 空字符串、空列表 |
| 全相同 | `[1, 1, 1]` |
| 全不同 | `[1, 2, 3]` |
| 单调递增 | `[1, 2, 3, 4]` |
| 单调递减 | `[4, 3, 2, 1]` |
| 刚好等于 | `sum == target` |
| 下标边界 | `l = 0`, `r = n - 1` |

## 为什么边界重要

你的主体思路可能对，但边界错会直接 WA。

例如前缀和：

```python
prefix[r + 1] - prefix[l]
```

如果 `r = n - 1`，那么 `r + 1 = n`，所以 `prefix` 必须有 `n + 1` 个位置。

## 自测样例习惯

不要只测官方样例。自己造：

```text

n = 2
全一样
最大值
没有答案
刚好一个答案
```
