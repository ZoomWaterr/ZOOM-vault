# 我现在该干啥

## 先看这个

你卡住时先别急着写代码。先问一句：**题目到底要我把输入变成什么输出？**

## 四步定位

| 你脑子里的问题 | 先去哪里 | 最常见动作 |
|:---|:---|:---|
| 输入怎么接 | [[../10-Python语法速查/01-输入输出]] | `input().split()` / `list(map(int, ...))` |
| 要统计次数 | [[../10-Python语法速查/07-字典与计数]] | `count[x] = count.get(x, 0) + 1` |
| 要找最大最小 | [[../10-Python语法速查/05-列表]] | `max` / `min` / `key=` |
| 要处理连续区间 | [[../30-算法路线与模板/03-前缀和与差分]] | 前缀和、差分、滑动窗口 |
| 要排序后处理 | [[../30-算法路线与模板/02-排序与二分]] | `sort`、`sorted`、二分 |
| 要一层一层尝试 | [[../30-算法路线与模板/06-搜索DFS与BFS]] | DFS/BFS/回溯 |

## 现场决策口诀

1. 先把输入读对。
2. 用最笨方法想一遍，不急着优化。
3. 看数据范围，判断笨方法会不会炸。
4. 写完先跑边界，不要马上交。

## 别想复杂

新手阶段最容易把题想成“必须有高级算法”。很多基础题其实就是：读入、循环、判断、计数、输出。先把这五件事串顺，算法会自然接上来。
