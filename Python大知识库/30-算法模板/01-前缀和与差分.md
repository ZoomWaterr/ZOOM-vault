# 前缀和与差分

---

## 前缀和 ⭐

### 💡 一句话理解

前缀和就是**从头一直加**。预先算好 `pre[i]` = 前 i 个元素的和，之后问「区间 [L,R] 的和」直接 O(1) 秒答。

### 🧠 怎么来的

如果你要反复查询"第 2 个到第 5 个元素之和"，每次都重新加一遍太慢了。不如预先算一个数组，`[L,R]` 的和 = `pre[R] - pre[L-1]`，一次减法搞定。

```
演示：
原数组：  [1]  [2]  [3]  [4]  [5]
          ↓    ↓    ↓    ↓    ↓
前缀和：  [1]  [3]  [6]  [10]  [15]
          1   1+2  3+3  6+4  10+5

问：下标 1~3 的和（2+3+4=9）
答：pre[3] - pre[0] = 10 - 1 = 9 ✓
```

### 📝 代码模板

```python
# ── 一维前缀和（原地修改，省空间）──
nums = [1, 2, 3, 4, 5]
for i in range(1, len(nums)):
    nums[i] += nums[i - 1]
# nums → [1, 3, 6, 10, 15]

# ── 查询区间 [L, R] 的和（0 基下标）──
def 区间和(pre, L, R):
    if L == 0:
        return pre[R]
    return pre[R] - pre[L - 1]
```

### 🎯 什么时候用

- 多次查询区间和
- "子数组和为 K" → 前缀和 + 哈希表
- 配合差分数组（前缀和还原差分）

### 🏋️ 做过

- 1501 分苹果

## 差分数组 ⭐⭐

### 💡 一句话理解

差分记录的是**变化量**。区间批量加减，只需要改两个位置。最后前缀和还原出真实值。

**"一个范围内，批量做同一件事" → 区间操作 → 想差分！**

### 🧠 怎么来的

如果 100 个学生，给第 20~50 名每人 +5 分，你会真的循环改 31 次吗？
更聪明：第 20 名标记"+5"，第 51 名标记"-5"。最后从头累加标记，还原出真实成绩。

```
演示：5 个小朋友，第 2~4 个每人 +3 颗糖

差分数组初始化： [0, 0, 0, 0, 0, 0]
                      ↑           ↑
                 在 L=2 +3   在 R+1=5 -3
                     ↓           ↓
打标记后：       [0, 0, +3, 0, 0, -3]

前缀和还原（从头累加）：
                [0] → [0] → [3] → [3] → [3] → [0]
                              第2~4位 ↑────↑
```

### 📝 代码模板

```python
# ── 模板：m 次区间加法，最后输出结果 ──
n, m = map(int, input().split())

# 第一步：差分数组（多开一位防 R+1 越界）
d = [0] * (n + 2)

# 第二步：打标记 — 每次只改两个位置，O(1)
for _ in range(m):
    L, R, C = map(int, input().split())
    d[L] += C              # 从 L 开始 +C
    d[R + 1] -= C          # 到 R 结束，R+1 之后抵消

# 第三步：前缀和还原
for i in range(1, n + 1):
    d[i] += d[i - 1]

# 第四步：输出（下标 1~n）
print(*d[1:n + 1])
```

### ⚠️ 易错点

- 差分数组多开 2 位防止 `R+1` 越界
- 标记是 `L` 加、`R+1` 减，不是 `R`！
- 最后一定要前缀和还原，差分数组本身不是答案

### 🎯 什么时候用

"区间批量加/减"、"一个范围内批量做同一操作" → 立刻想差分

### 📊 复杂度对比

| 方法 | 每次区间操作 | 最后查询 | 总复杂度 |
|:---|:---|:---|:---|
| 暴力 | O(n) 每次 | O(1) | O(m × n) |
| 差分 | O(1) 每次 | O(n) 一次 | O(m + n) |

### 🏋️ 做过

- 1501 分苹果

## 💬 德远的笔记

> 差分数组的核心思想就一句话：**与其真的去填满 L 到 R，不如只打两个标记**——"L 这里开始多了 C"和"R+1 这里少了 C"。最后统一结算（前缀和）。
>
> 第一次接触「分苹果 (1501)」那道题的时候，暴力想法就是给每个小朋友循环加苹果——但 M 次操作每次都 O(n)，总复杂度 O(m×n) 直接超时。用差分之后变成 O(m+n)，快了不知道多少倍。
>
> 这两个东西（前缀和 + 差分）经常一起出现：前缀和用来查询，差分用来修改。把这两个模板背下来，区间类的题基本不怕了。
