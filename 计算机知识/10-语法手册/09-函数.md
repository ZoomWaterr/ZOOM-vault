# 函数

> 把一段代码打包，给它一个名字，以后叫名字就能用。

---

## 定义与调用

```python
# 定义
def 函数名(参数列表):
    函数体
    return 返回值

# 调用
函数名(参数)
```

- 必须先定义，再使用
- 定义时不执行，调用时才执行
- 函数中通过缩进描述归属关系

---

## 参数传递方式

```python
def reg_stu(name, age, gender, city):
    print(f"注册：{name}, {age}, {gender}, {city}")
    return {"name": name, "age": age, "gender": gender, "city": city}

# 位置参数 — 按顺序对应
s = reg_stu("德远", 18, "男", "北京")

# 关键字参数 — 指定参数名，顺序无关
a = reg_stu(name="德远", age=18, gender="男", city="北京")
```

## 默认参数

```python
def reg_stu(name, age, gender, city="北京"):    # city 有默认值
    ...

# 调用时可以省略有默认值的参数
reg_stu("张三", 20, "男")              # city 用默认的"北京"
reg_stu("李四", 22, "女", "上海")     # 覆盖默认值
```

> ⚠️ 默认参数必须放在普通参数**之后**！

## 不定长参数

```python
def just_do(*args, **kwargs):
    # *args   → 接收多余的位置参数，封装成元组
    # **kwargs → 接收多余的关键字参数，封装成字典
    ...

# 调用
just_do(10, 20, 30, round=2, count=0)
# args = (10, 20, 30)
# kwargs = {"round": 2, "count": 0}
```

## 类型注解

```python
def is_prime(n: int) -> bool:
    """判断 n 是不是素数"""
    if n < 2:
        return False
    for i in range(2, int(n ** 0.5) + 1):
        if n % i == 0:
            return False
    return True
```

> 类型注解不影响运行，但对阅读和维护很有帮助。

## 元组解包 — 函数返回多个值

```python
def min_max(arr):
    return min(arr), max(arr)          # 返回的其实是 (min, max) 元组

最小值, 最大值 = min_max([3, 1, 4, 1, 5])   # 解包接收
```

拓展解包：

```python
a, *b, c = (9, 2, 34, 123)
# a = 9, b = [2, 34], c = 123
```

## 💬 德远的笔记

> 诊断报告说我 86% 的代码是平铺的，只在递归和质数判断时才被迫写函数。确实——刷题的时候总觉得「写函数多此一举」，但后来发现把核心逻辑抽成函数有几个实在的好处：
> 1. 出 bug 时一眼就知道进哪个函数找，不用在 30 行平铺代码里翻
> 2. 质数判断写一次，三道题都能复制用
> 3. 主逻辑 3~5 行，脑子只装一件事
>
> 现在至少会写一个 `def solve():` 把主逻辑包起来。不费事但效果好。
